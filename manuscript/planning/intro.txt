# Intro {#chapter-planning-intro}

Design flaws are as bad as programming bugs: Both are vulnerabilities
an attacker can abuse.

A software or hardware design well done can simplify programming and
strongly improve security.

Many specific technologies require domain knowledge to be used in a
secure fashion. And this is available but hidden under heaps of
interesting features those technologies offer.

This chapter will extend principles from the background chapter into
technology specific knowledge.
